# Historia de Usuario

## ID
[DB-01]

## Título
Modelo de datos y migraciones con SQLAlchemy

## Descripción
**Como** equipo de desarrollo

**Quiero** tener el esquema relacional definido y versionado desde el inicio del sprint

**Para** asegurar la integridad de los datos, facilitar el trabajo colaborativo y permitir despliegues consistentes mediante migraciones.

## Criterios de Aceptación

### Escenario 1: Definición del Esquema ORM
```gherkin
Dado que el proyecto utiliza SQLAlchemy como ORM
Cuando se definen los modelos "User", "Course" y "Application"
Entonces cada tabla debe incluir sus claves primarias, foráneas (FK) y relaciones "backref"
Y los timestamps (created_at, updated_at) deben generarse automáticamente por el servidor
```

### Escenario 2: Implementación de Enums para Roles y Estados
```gherkin
Dado que el sistema requiere estados y roles específicos
Cuando se define el campo "role" en la tabla "User"
Entonces debe restringirse a los valores: "user", "admin" y "superadmin"
Y el campo "status" en la tabla "Application" debe ser: "pending", "accepted", "rejected" o "cancelled"
```

### Escenario 3: Versionado de Base de Datos (Alembic)
```gherkin
Dado que el esquema puede evolucionar durante el sprint
Cuando se ejecuta el comando "alembic upgrade head"
Entonces la base de datos debe actualizarse a la última versión disponible
Y el historial de migraciones debe guardarse en el repositorio de Git
```

### Escenario 4: Poblamiento Inicial (Seed) y Documentación
```gherkin
Dado un entorno de base de datos recién migrado
Cuando se ejecuta el script "seed.py" (vía "python scripts/seed.py")
Entonces el sistema debe insertar 2 admins, 5 users, 3 cursos y 10 solicitudes de prueba
Y el diagrama ERD debe estar visible en el README para consulta del equipo
```

## Notas

- **Contexto de FastAPI**: Usar request.state para persistir la identidad del usuario durante el ciclo de vida de la petición.
- **Jerarquía**: superadmin hereda permisos de admin, pero admin no hereda permisos de superadmin.
- **Enums**: Se deben usar constantes de SQLAlchemy para evitar errores de escritura en la lógica de comparación.
- **Contexto**: La configuración de Alembic debe apuntar dinámicamente a la variable de entorno DATABASE_URL.
- **Seguridad**: El error 403 debe ser genérico para evitar la enumeración de roles o rutas existentes.
