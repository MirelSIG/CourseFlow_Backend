# Historia de Usuario

## ID
[DB-02]

## Título
Configuración del entorno y variables

## Descripción
**Como** equipo de desarrollo

**Quiero** un entorno reproducible con configuración separada para desarrollo (dev) y producción.

**Para** asegurar la seguridad de las credenciales y facilitar el despliegue en diferentes infraestructuras sin modificar el código fuente.

## Criterios de Aceptación

### Escenario 1: Gestión de entornos mediante clases
```gherkin
Dado que el sistema arranca en diferentes contextos
Cuando se lee la variable de entorno "ENVIRONMENT"
Entonces el sistema debe cargar la clase "DevelopmentConfig" o "ProductionConfig" según corresponda
Y aplicar las configuraciones de base de datos y depuración específicas.
```

### Escenario 2: Seguridad de credenciales y plantillas
```gherkin
Dado que el proyecto utiliza Git para el control de versiones
Cuando el desarrollador revisa los archivos del repositorio
Entonces el archivo ".env" debe estar excluido mediante ".gitignore"
Y debe existir un archivo ".env.example" con las claves obligatorias vacías o de ejemplo.
```

### Escenario 3: Compatibilidad de motores de Base de Datos
```gherkin
Dado que se trabaja con SQLite en local y PostgreSQL en producción
Cuando se configura "DATABASE_URL"
Entonces SQLAlchemy debe ser capaz de conectar con ambos motores de forma transparente basándose en el string de conexión.
```

## Notas

- **Contexto de FastAPI**: Se utilizará la librería python-dotenv para cargar automáticamente las variables al iniciar la aplicación.

- **Jerarquía**: La configuración de producción debe tener por defecto DEBUG = False y requerir obligatoriamente una SECRET_KEY compleja.

- **Enums**: Las variables de entorno para tiempos de expiración de JWT (ej: JWT_EXPIRATION_DELTA) se convertirán a enteros para su uso en la lógica.

- **Contexto**: config.py centralizará todas las llamadas a os.getenv() para evitar que las variables estén dispersas por el código.

- **Seguridad**: Si faltan variables críticas en producción (como SECRET_KEY), la aplicación debe detener el arranque (fail-fast).
